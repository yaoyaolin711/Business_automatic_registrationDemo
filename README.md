# 工商注册流程 AI 化 Demo

可演示的本地 Demo：模拟企微对接 → 材料确认 → 本地建档 → Mock CR 申请账号 → 邮箱取号 → Playwright 自动填报 → 通知人工。

## 快速开始

```bash
cd "工商注册流程自动化Demo"
python -m pip install -r requirements.txt
python -m playwright install chromium
python run.py
```

浏览器打开：http://127.0.0.1:8787

## 演示步骤

1. 点击「新建企微群案件」——机器人发送材料清单  
2. 可在对话框问「可以用大陆手机号吗？」等 FAQ  
3. 点击「填充演示数据」——生成确认单  
4. 可点「填前校验」查看业务完整性（持股合计、必填项等）  
5. 点击「确认提交」  
6. 点击「正常自动流程」——归档、字段映射填表、回读校验、读邮箱、正式填报  
7. 也可点「演示漏填拦截 / 演示错填拦截」看系统如何阻断提交  
8. 右侧查看流程看板、字段映射、校验报告、截图  
9. 点击「人工确认完成」结束

## 防漏填 / 错填机制

1. **字段映射**：按 `#id` / `data-field` 填，不靠屏幕坐标  
2. **填前校验**：公司名、业务性质≤30字、持股合计100% 等  
3. **填后回读**：把页面值读回与期望比对，不一致则**阻断提交**  
4. **报告落盘**：`screenshots/<案件ID>/validation_*.json`，归档目录同步一份  

## 目录说明

| 路径 | 作用 |
|------|------|
| `app/` | FastAPI、状态机、归档、邮箱解析、RPA |
| `static/` | 演示前端 + Mock CR 页面 |
| `archives/` | 按公司名自动建档 |
| `mock_mailbox/` | 模拟邮箱信件 |
| `screenshots/` | Playwright 截图 |
| `cases/` | 案件 JSON |

## 说明

- CR 官网用本地 Mock 页替代，避免验证码/登录限制，便于稳定演示。  
- 真实环境可将 RPA 目标替换为 ICRIS 正式地址，并增加人工验证码节点。
